# Generated-wire reference slice

The second schema reference tier: the same wire/domain discipline as
[`examples/Axial.ReferenceApp`](../Axial.ReferenceApp/), but the wire tier is generated from `[<DeriveSchema>]`
records instead of hand-written builder pipelines.

## Run it

```bash
dotnet run --project examples/Axial.ReferenceApp.Wire/Axial.ReferenceApp.Wire.fsproj --nologo
```

## Layout

- `wire/workspace.fs` — user-owned records with constraint attributes (`Min`, `Max`, `Email`, `Distinct`,
  `SchemaName`, `Default`), a `SchemaConstructor` static member normalising the owner email on parse, and
  the `WorkspaceCardV1`/`WorkspaceCard` version-chain naming convention.
- `wire/workspace.g.fs` — generated by `axial schemagen`: schemas, `parse`/`validate`, and the
  `WorkspaceCard.contract` builder that takes the hand-written migration.
- `Program.fs` — the only hand-written parts: the fallible v1 → v2 migration, a strict domain mapping
  (`TrustedCard`) enforcing a rule the wire cannot, a head-version write through a compiled `Axial.Schema.Json`
  codec, and `JsonSchema.generate` output from the generated schema.

## Regenerating

Package consumers reference `Axial.Schema.Contracts.Build` (with `PrivateAssets="all"`). The build discovers
`[<DeriveSchema>]` in ordinary F# compile files and generates under `obj` by default. This reference app sets
`AxialSchemaGeneratedFiles` to `CheckedIn`, so an ordinary build refreshes `wire/workspace.g.fs` after edits.
The equivalent direct command is:

```bash
dotnet run --project scripts/schemagen -- examples/Axial.ReferenceApp.Wire/wire
```

The generated module constructs the record by name, so a stale `.g.fs` fails to compile instead of drifting.
